漏洞评估

漏洞评估的目的是找出最有可能攻击的安全漏洞。在杀链的利用阶段，漏洞评估的重点是创建访问以实现漏洞的目标映射，以排列漏洞和保持对目标的持续访问

1. 漏洞命名

漏洞的扫描采用自动化过程和自动化程序。漏洞扫描会被检测到，隐形几乎不可能的。而且还会收到以下限制：

  • 通常情况下，漏洞扫描器是基于签名的，它只能检测已知的漏洞，仅当有一个已经存在的识别标志才能应用于目标
  • 扫描器产生的大量输出中通常包含错误结果
  • 扫描器可能对网络有负面影响，可能造成网络延迟或某些设备失效
  • 在某些地区，扫描被认为是黑客行为，可能构成违法

2 本地和在线漏洞数据库

漏洞的搜索可以从供应商的网站开始。同事在发布补丁和升级时，大多数的硬件和应用程序供应商都会提供和安全漏洞相关的信息

收集、分析和共享有关漏洞的网站信息如下：

  • 国家漏洞数据库，整合了所有由美国政府发布的公开漏洞数据：http://web.nvd.nist.gov/view/vuln/search
  • 安全监控网站Secunia：http://secunia.com/community/
  • 数据包风暴安全网址：http://packetstormsecurity.com/
  • 安全焦点网址：http://www.securityfocus.com/vulnerabilities
  • 由Offensive Security维护的可以利用的漏洞数据库网址：http://www.exploit-db.com
  • 渗透测试人员可以浏览一下https://0day.today/获取零日漏洞信息

漏洞数据库也被复制到本地Kali中，并且可以在/usr/share/exploitdb目录中找到

搜索漏洞数据库的本地副本，需要使用如下命令searchsploit 关键词。这将调用一个脚本搜索所有漏洞列表的数据库文件(.csv)，返回已知漏洞的描述，以及和该漏洞相关的路径

搜索脚本从左至右扫描数据库CSV文件中的每一行，因此搜索项的顺序很重要，如：搜索Oracle10g将返回多个漏洞，但10g Oracle本身不会返回任何漏洞、搜索“vsFTPd”不会返回任何结果，而“vs FTPd”（“vs”和“FTPd”中间有空格）则会返回很多结果

漏洞利用以脚本的形式呈现，例如：Perl、Ruby和PHP等，都是相对容易实现的脚本

但是还有不易编译的脚本，且可能存在危险的漏洞利用：

  • 蓄意的错误或不完整的源代码，因此有经验的开发人员尽量让漏洞远离没有经验的用户
  • 漏洞利用并不总是有详尽的文档解释
  • 由于环境变化导致行为不一致（新的补丁应用到目标系统、目标应用程序中的语言变化）可能需要对源代码进行重大调整，这需要一个有经验的开发人员
  • 免费提供的、包含恶意的代码，总是存在风险。渗透测试人员可能会认为它是在进行概念验证（POC）练习，并不知道该代码同时为该应用程序建了一个后门，可以供开发者使用

3. 用nmap进行漏洞扫描

nmap还可以用于执行漏洞评估，可以利用nmap的功能编写自己的脚本，并使其自动化以方便利用，开发NSE的主要原因如下：

  • 网络发现：攻击者利用nmap的主要目的是网络发现
  • 服务的分类版本检测：共计数千种服务。把多个版本的详细信息集中到同一个服务上显得更加复杂
  • 漏洞检测：在广泛的网络范围内自动识别漏洞，然而，nmap本身不完全是漏洞扫描程序
  • 后台检测：一些脚本被编写用来识别后台模式，如果网络上有任何蠕虫感染，一些脚本将被写入以识别该模式，这样使得攻击者的工作变得更集中，能专注于远程接管机器

nmap更新脚本数据库命令nmap --script-updatedb

针对目标主机运行脚本nmap -T4 -A -sV -v3 -d -oA Target output --script all --script-args vulns.showall target.com

3.1 Lua脚本介绍

Lua是一种建立在C语言之上的轻量级嵌入式脚本语言，于1993年在巴西创立，至今仍然在积极发展中。它是一种强大而快速的编程语言，主要用于游戏应用和图像处理。Lua开发的安全工具有nmap、Wireshark和Snort 3.0等

4. Web应用漏洞扫描器

漏洞扫描器具有所有扫描器沟通的缺点(只能检测已知漏洞的签名，且误报率较高)

Kali针对Web服务器配有大量漏洞扫描器，并为安装新的扫描器和扩展其能力提供了稳定的平台。测试人员可以通过选择扫描工具来提高测试的有效性：

  • 最大限制的完整性和测试精度
  • 尽量减少获得可用的结果所需的时间
  • 尽量减少对被检测网络服务的负面影响

Kali提供几种不同的漏洞扫描器，主要如下：

  • 扩展了传统漏洞扫描程序的功能，包括网站和关联服务(例如：Metasploit框架和Websploit)的扫描器
  • 拓展了非传统的应用功能，如Web浏览器，支持Web服务漏洞扫描(OWASP Mantra)的扫描器
  • 专门开发用于支持对网站和网页服务的侦察和利用检测(Arachnid、Nikto、Skipfish、Vega、w3af等)的扫描器

4.1 Nikto和Vega简介

Nikto是一个简单的扫描器，通常提供准确但有限的结果

使用命令nikto -h IP/URL -p Port

基于验证发现漏洞数量的、最有效的扫描器之一是Subgraph Vega，它会扫描目标，并将漏洞分级为高、中、低和信息性的。为了得到特定的结果，该测试器能够点击确定结果为“drill down”（深入）。该测试器还可以修改搜索模块，这是用Java编写的，专注于特定的漏洞或找出新的安全漏洞

5. 移动应用漏洞扫描器

本节中介绍设置移动应用程序扫描器，以及如何将移动应用程序扫描器的结果组合在一起，并利用信息来识别更多的漏洞，以实现渗透测试的目标

移动安全框架(Mobile Security Framework, MobSF)是针对所有移动平台(包括Android、IOS和Windows)的开源自动渗透测试框架。框架是用Djiango Python框架编写的

框架下载地址：https://github.com/MobSF/Mobile-Security-Framework-MobSF

解压后进入对应的文件夹下，输入以下命令python3 -m pip install -r requirements.txt

一切准备就绪后，使用命令python3 manage.py test来测试配置

转移应用程序安装python manage.py migrate

漏洞扫描命令python3 manage.py runserver 192.168.66.1:8000

在浏览器中访问http://yourIPaddress:Portnumber，将侦察期间发现的任何移动应用程序上传到扫描器，用于识别入口点。

一旦文件上传，渗透测试人员可以识别扫描器的配置文件以及所有其他重要信息!

6. 网络漏洞扫描器OpenVAS

开放漏洞评估系统(Open Vulnerability AssessmentSystem，OpenVAS)是一个开源的漏洞评估扫描器，也是攻击者通常用来大规模扫描网络的漏洞管理工具，其中大约包括有47 000个漏洞的数据库

安装命令apt-get install openvas

安装完成后，运行openvas-setup(现在OpenVAS更名为了GVM（Greenbone Vulnerability Management）命令同样更新为gvm-setup)命令安装依赖OpenVAS。为了确保安装可行，可以运行openvas-check-setup(更新为gvm-check-setup)命令，它会列出有效运行OpenVAS所需的前10个项目

来设置管理员用户密码信息会在gvm-setup命令的末尾显示

gvm-feed-update更新数据库

并通过openvar-start(gvm-start)命令启动。启动后访问https://localhost:9392端口即可看到服务命令`gvm-stop`停止服务

登录信息是刚才设置的内容

如果想不通过localhost来进行访问参考链接

实际操作流程如下即可

vi /lib/systemd/system/greenbone-security-assistant.service

将其中127.0.0.1的部分修改为0.0.0.0，然后保存

systemctl daemon-reload
启动服务

gvm-start

与其他扫描器不同的是，OpenVAS还可以自定义扫描配置，它允许测试人员添加凭证，禁用特定的插件，并设置可以进行的最大和最小连接数等

7. 商业漏洞扫描器

商业漏洞扫描器此处主要介绍Nessus和Nexpose

此处仅简单介绍安装流程，具体内容请参考对应的综合文档

7.1 Nessus

在Kali上的安装步骤为：

  1. 访问https://www.tenable.com/try网站并注册为普通用户，然后选择Try Nessus Professional Free（免费试用Nessus专业版）
  2. 从https://www.tenable.com/downloads/网站下载对应的Nessus版本
  3. 下载完成后，通过以下命令进行安装：dpkg -i Nessus-8.1.2-debian6_amd64.deb
  4. 接着通过运行service nessusd start开启nessus服务，然后Nessus就在系统中启动了
  5. 默认情况下，Nessus扫描器的运行端口是SSL的8834号端口
  6. 添加一个新用户并激活注册码。扫描器将根据注册码下载所有相关插件
  7. 最后，将看到Nessus的运行界面，此时可以对目标系统或网络实施扫描了

7.2 Nexpose

在Kali上的安装步骤为：

  1. 注册Rapid 7账号（https://www.rapid7.com/products/nexpose/request/），需要一个有效的商业电子邮箱来接收激活码
  2. 运行以下代码，从官网下载安装包wget http://download2.rapid7.com/download/InsightVM/Rapid7Setup-Linux64.bin之后赋予编译文件权限，根据扫描器要求输入信息即可
  3. 默认情况下，Nexpose运行端口为SSL的3780号端口。所以能通过https://localhost:3780/访问Nexpose
  4. 同样，Nexpose也会根据注册码下载所有插件，随后可以登录Nexpose漏洞扫描器了

9. 威胁建模

威胁建模：最初是由网络规划者发展起来的，制定针对攻击的防御对策的过程

渗透测试人员需要不断寻找防御威胁建模方法来提高攻击的成功率。攻击威胁建模是制定攻击策略的一个较为常见的手段，它将侦察和研究的结果相结合。攻击者必须充分考虑现有的目标，并确定目标是以下何种类型：

  • 首要目标：这些是任何组织的入口点目标。当这些目标被击垮时，它们将直接帮助攻击者达到目的
  • 次要目标：这些目标可以提供信息（安全控制、密码、登录政策、以及本地和域管理员的用户名和密码）以支持对首要目标进行攻击或访问
  • 第三目标：这些目标可能与测试或攻击对象无关，但是相对容易攻击，有可能为实际攻击提供信息或者其他思路
